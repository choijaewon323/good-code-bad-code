# 5.7 설명되지 않은 값을 사용하지 말라

- 하드코딩한 값이 무엇을 의미하는지 다른 개발자들이 명확히 이해하도록 하는 것을 자주 잊는다

## 5.7.1 설명되지 않은 값은 혼란스러울 수 있다

```java

class Vehicle {
    double getKineticEnergyJ() {
        return 0.5 * getMassUsTon() * 907.1847 * Math.pow(getSpeedMph() * 0.44704, 2);
    }
}

```


- 각 숫자가 무엇을 의미하는지 잘 안보인다
- 만약 다른 개발자가 getMassUsTon() 함수를 없애고 질량을 킬로그램으로 반환하는 getMassKg() 함수로 변환한다면?
  - 907.1847 이 미국 톤을 킬로그램으로 변환하는 계수라는 걸 이해하지 못하여 잘못된 값이 나옴

## 5.7.2 해결책: 잘 명명된 상수를 사용하라

```java
class Vehicle {
    private static final double KILOGRAMS_PER_US_TON = 907.1847;
    private static final double METERS_PER_SECOND_PER_MPT = 0.44704;

    double getKineticEnergyJ() {
        return 0.5 * getMassUsTon() * KILOGRAMS_PER_US_TON * Math.pow(getSpeedMph() * METERS_PER_SECOND_PER_MPH, 2);
    }
}
```

- 의미가 더해졌다.

## 5.7.3 해결책: 잘 명명된 함수를 사용하라

- 공급자 함수 : 상수를 반환
- 헬퍼 함수 : 변환을 수행

공급자 함수
```java
class Vehicle {
    double getKineticEnergyJ() {
        return 0.5 * getMassUsTon() * kilogramsPerUsTon() * Math.pow(getSpeedMph() * metersPerSecondPerMph(), 2);
    }

    private static double kilogramsPerUsTon() {
        return 907.1847;
    }

    private static double metersPerSecondPerMph() {
        return 0.44704;
    }
}

```

헬퍼 함수
- 계수에 대한 건 사실 구현 세부 사항이므로 함수 호출쪽에서는 몰라도 되는 정보일 수 있다.
```java
class Vehicle {
    double getKineticEnergyJ() {
        return 0.5 *  * kilogramsPerUsTon(getMassUsTon()) * Math.pow(metersPerSecondPerMph(getSpeedMph()), 2);
    }

    private static double kilogramsPerUsTon(double usTons) {
        return usTons * 907.1847;
    }

    private static double metersPerSecondPerMph(double mph) {
        return mph * 0.44704;
    }
}

```

- 만약 헬퍼 함수나 정의한 값이 다른 개발자들이 사용한다면, 유틸리티 클래스에 두는 것이 좋다.
