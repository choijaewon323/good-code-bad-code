# 5.6 함수 호출도 가독성이 있어야 한다

- 함수 인수가 많으면 이해하기 힘들다

## 5.6.1 매개변수는 이해하기 어려울 수 있다

```java
sendMessage("hello", 1, true);
```

- 이 코드가 있을 때, 1과 true는 무슨 뜻인지 명확하지 않음
- 이걸 알려면, 함수 정의를 살펴봐야만 함

## 5.6.2 해결책: 명명된 매개변수 사용

```java
sendMessage(message: "hello", priority: 1, allowRetry: true);
```
- 인자의 위치가 아닌 이름으로 일치하는 매개변수 찾는 기능으로 가독성 향상 가능
- 하지만 안되는 언어도 있음
- 타입스크립트도 기본적으로는 안되고, 인터페이스 선언해서 집어넣는 방법이 있긴 함
- 자바는 어떻게?

## 5.6.3 해결책: 서술적 유형 사용

- 정수와 불리언 값으로는 의미가 불분명함
- 아예 이를 위한 클래스나 enum을 만들어 의미를 더함

```java
class MessagePriority {}

enum RetryPolicy {
    ALLOW_RETRY, DISALLOW_RETRY
}

void sendMessage(
    String message,
    MessagePriority priority,
    RetryPolicy retryPolicy
) {}

sendMessage("hello", new MessagePriority(1), RetryPolicy.ALLOW_RETRY);
```

## 5.6.4 때로는 훌륭한 해결책이 없다

```java
class BoundingBox {
    BoundingBox(int top, int right, int bottom, int right) {}
}
```

- 명명된 매개변수를 지원 안하면, 숫자만 보여주므로 이해가 어려움
- 인라인 주석을 사용할수도 있으나, 이것도 관리 포인트가 됨을 주의해야함
- setter나 빌더 패턴이 가능하나, 값이 누락될 수 있음을 유의해야함

## 5.6.5 IDE는 어떤가?

- IntelliJ 와 같은 IDE는 이런걸 잘 보여주고 있다.
- 하지만 이 기능에 의존하지 않는 것이 좋음
  - 다른 IDE를 사용할 수도 있기 떄문
