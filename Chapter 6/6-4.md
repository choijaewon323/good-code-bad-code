# 6.4 입력 매개변수를 수정하는 것에 주의하라

## 6.4.1 입력 매개변수를 수정하면 버그를 초래할 수 있다

- 객체를 매개변수로 받는데, 그 객체를 수정하는 경우 버그 발생 가능

## 6.4.2 해결책: 변경하기 전에 복사하라

```java

List<Invoice> getBillableInvoices(
    Map<User, Invoice> userInvoices,
    Set<User> usersWithFreeTrial
) {
    return userInvoices
    .entries()
    .filter(entry -> !userWithFreeTrial.contains(entry.getKey()))
    .map(entry -> entry.getValue());
}

```
- 변경 전에 새 자료구조에 복사하여 변경했다
- 만약 너무 오버헤드가 심하면 -> 함수 이름과 문서에 이러한 수정이 발생하는 걸 분명히 해야함
