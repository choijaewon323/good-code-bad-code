# 6.6 미래를 대비한 열거형 처리

## 6.6.1 미래에 추가될 수 있는 열것값을 암묵적으로 처리하는 것은 문제가 될 수 있다

```java

enum PredictedOutcome {
    COMPANY_WILL_GO_BUST,
    COMPANY_WILL_MAKE_A_PROFIT
}

boolean isOutcomeSafe(PredictedOutcome prediction) {
    if (prediction == COMPANY_WILL_GO_BUST) {
        return false;
    }
    return true;
}

```

- 열거형이 2개일때는 괜찮은데, 나중에 더 추가되면 문제가 발생할 수 있음

## 6.6.2 해결책: 모든 경우를 처리하는 스위치 문을 사용하라

- 모든 열것값을 명시적으로 처리하고, 새로 추가되는 경우 컴파일 실패 혹은 테스트가 실패하도록 만들어야함

```java

enum PredictedOutcome {
    COMPANY_WILL_GO_BUST,
    COMPANY_WILL_MAKE_A_PROFIT
}

boolean isOutcomeSafe(PredictedOutcome prediction) {
    switch (prediction) {
        case COMPANY_WILL_GO_BUST:
            return false;
        case COMPANY_WILL_MAKE_A_PROFIT:
            return true;
    }
    throw new Exception("unhandled prediction");
}

```

## 6.6.3 기본 케이스를 주의하라

- 기본 케이스를 잘못 설정하면 위에서 발생했던 문제와 동일한 문제 발생 가능
- 예외 발생하게 하는게 가장 안전한듯
