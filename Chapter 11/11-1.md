# 11.1 기능뿐만 아니라 동작을 시험하라



## 11.1.1 함수당 하나의 테스트 케이스만 있으면 적절하지 않을 때가 많다
- public api 하나당 하나의 테스트 케이스만 있으면 올바르게 동작하는지 확인이 어렵다

## 11.1.2 해결책: 각 동작을 테스트하는 데 집중하라

모든 동작이 테스트되었는지 거듭 확인하라
- 삭제해도 컴파일되거나 테스트 통과되는 코드가 있는지?
- if 문의 참 거짓 논리를 반대로 해도 테스트가 통과되는지?
- 논리 연산이나 산술 연산을 다른걸로 해도 테스트가 통과되는지?
- 상숫값이나 하드 코딩 값을 변경해도 테스트 통과되는지?
  - 위 네개에 대한 대답이 예 라면 모든 행동이 테스트되지 못하는 것
- 예외도 예외 종류나 메시지가 제대로 나오는지 테스트할 필요 있음
