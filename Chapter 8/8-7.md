# 8.7 예외 처리 시 구현 세부 사항이 유출되지 않도록 주의하라

## 8.7.1 예외 처리 시 구현 세부 사항이 유출되면 문제가 될 수 있다

- 하위 계층의 예외를 상위에서 catch하는 경우, 하위 계층의 구현 상세 로직을 알게 되어 catch 로직이 쓸모없어질 수 있다
- 또한 하위 계층의 세부사항을 상위가 아는 것 또한 좋은 설계가 아님

## 8.7.2 해결책: 추상화 계층에 적절한 예외를 만들라
- 하위 계층에서 상위 계층에 맞는 예외로 래핑하여 던진다
- 만약 예외로부터 복구하지 않을거라면 굳이 추상화해서 다시 던질 필요는 없음
