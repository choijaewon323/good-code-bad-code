# 7.3 지나치게 일반적인 데이터 유형을 피하라

## 7.3.1 지나치게 일반적인 유형은 오용될 수 있다

- 지도상에서의 위치를 나타낼 떄 위경도를 주로 쓰는데, 리스트로 구현할 수 있다
- 그러나 너무 일반적이여서, 코드를 오용하기 쉽다
  
```java

class LocationDisplay {
    private final DrawableMap map;

    void markLocationsOnMap(List<List<Double>> locations) {
        for (var location : locations) {
            map.markLocation(location[0], location[1]);
        }
    }
}

```

- 문제점
  - 주석문으로 어떻게 값을 넣어야 하는지 알려줘야함
  - 순서도 헷갈림 - 위경도인지, 경위도인지


## 7.3.2 페어 유형은 오용하기 쉽다

- 위와 동일한 문제 발생

## 7.3.3 해결책: 전용 유형 사용

- 새로운 클래스를 만들어주자
```java
class LatLong {
    private final Double latitude;
    private final Double longitude;

    LatLong(Double lat, Double lng) {
        this.latitude = lat;
        this.longitude = lng;
    } 
}

void markLocationsOnMap(List<LatLong> locations) {
    for (var location : locations) {
        map.markLocation(
            location.getLatitude(),
            location.getLongitude()
        )
    }
}
```

- 명확해졌다.
